Hexo添加多说评论框指南

本文关于如何为Hexo添加多说评论框

Hexo默认使用的第三方评论系统是Disqus，Disqus是外国的评论系统，貌似挺红火的，不过听说不太符合天朝的国情，不支持什么新浪微博啊之类的吧？还是把它换成多说比较接地气。

安装过程

STEP 1

前往多说注册，添加自己的网站信息，就会获得类似以下的HTML代码。

STEP 2

找到自己使用的theme的文件夹里面的某个名叫comment.ejs的文件，例如我的主题名称叫tyrant，那么comment.ejs位于的路径是themes\tyrant\layout_partial\comment.ejs，打开它并准备编辑。

STEP 3

删除comment.ejs原本的所有内容（注：原本的内容就是使用Disqus要用到的代码，可先备份下来）。
将STEP 1的代码复制到该文件中。

STEP 4

将多说提供的代码用以下两行代码包裹起来：
<% if (page.comments) { %>
这里要放置多说提供的代码
<% } %>

STEP 5

将 data-title=”请替换成文章的标题” 替换成 data-title=”<%= page.title %>”
将data-url=”请替换成文章的网址” 替换成 data-url=”<%= page.permalink %>”
将data-thread-key=”请将此处替换成文章在你的站点中的ID” 替换成data-thread-key=”<%= page.path %>”

编辑完后大概长成酱紫~

大功告成！！

原理浅析

首先，Hexo使用了EJS模板引擎。（什么是模板引擎？）

关键是第4和第5步。

第4步的作用是判断当前页面是否应该加入评论框框。如果不进行第4步，blog的每个页面都会添加上多说的评论框。它是通过page.comments的布尔值来判断的。至于哪个页面的page.comments应该设为什么样的值我们不需要关心，Hexo已经帮我们设好了。

至于第5步，主要就是告诉多说，哪条评论对应的究竟是你的博客里的哪篇文章。

step4和step5都用到了page这个object。

page的主要属性有：
{
“title”: “文章标题”,

“date”: “文章的发布日期（Moment.js 库）“,

“updated”: “文章的更新日期（Moment.js 库）“,

“comments”: “ 开启此文章的留言功能 boolean 值”,

“layout”: “ 文章布局”,
“content”: “文章内文”,

“excerpt”: “ 文章摘要（内文中 之前的内容）”,

“source”: “about/index.html 文件的相对路径”,

“path”: “about/index.html 文章的相对路径”,

“ctime”: “2013-04-05T14:38:36.000Z”,

“mtime”: “2013-04-05T14:38:41.000Z”,

“permalink”: “ 完整的网络访问路径”,

“full_path”: “D:\hexo11/source/about/index.html 原始文件路径”,

“categories” : “{}“,

“tags”: “{}”

“_id”: 1
}

ps.提供给多说的div属性data-thread-key，应该是用来作为每篇博文的唯一标识的，应该尽量保证它的唯一性，所以我选用了page.path

以上。
